SmartUse Admin

A backend to publish geospatial Data Packages- an application that allows us to connect and manage spatial information, then configure and export it to the SmartUse frontend application through an API based on the Frictionless Data standard.

Projects can be accessed and their models and viewed by clicking on the SmartUse logo on the home page. This is for the time beingi the primary navigation between projects. There is also a link on the home page to an online API platform for developers. The frontend is built on this basis, and other apps could also connect to SmartUse with the API.

The Web API can be used to access data about Projects and Resources.

User Guide

Note: you can use our online form at smartuse.ch/upload to send us PDF and other formats for processing. The following instructions detail how to manage your data publication in this backend.

On the home page there is a link to an Admin backend where projects and resources are defined.

Start by creating a User for every organization, with links to logos etc. The e-mail address is used to look for a Gravatar image.

Then create a Project which defines the study or location of interest. This is where the main documentation from the customer will reside. Use Markdown formatting.

Create a Resource for every visualization, map, image, or file shared with public visitors of the project. These will be shown above the project's description. Select the appropriate project, add a description and choose from one of the Dataformats supported:

  • png - a PNG (lossless) formatted image file
  • jpg - a JPEG (lossy) formatted image file
  • geojson - Web-optimized geospatial data - see http://geojson.org/
  • datapackage - a geospatial data package - see https://frictionlessdata.io/
  • embed - a Web-published map for use in an IFRAME

Data files for any of the formats above can be uploaded in the Data tab, and then linked with the path. For example, if you uploaded a GeoJSON file called pendlerorte.geojson into a test_project/data folder, you could just enter /uploads/test_project/data/pendlerorte.geojson.

Use the Features map to draw a polygon representing the location and shape of your data. In the future this will be automatically generated from supported formats and providers.

This project is being developed by the SmartUse ARGE team using GeoAlchemy, Flask and PostGIS.
